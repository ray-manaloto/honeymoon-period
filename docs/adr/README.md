# Architecture decision records

ADRs capture decisions that constrain future work. Research and specifications link to ADRs rather than copying them.

## Statuses

- **Proposed** — under review.
- **Accepted** — current policy.
- **Superseded** — replaced by a later ADR.
- **Deprecated** — retained for history but no longer recommended.

## Format

Use a zero-padded sequence and short slug. Include Status, Date, Context, Decision, Consequences, and Revisit when. Never silently rewrite an accepted decision; supersede it.
